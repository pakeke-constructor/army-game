

# Incremental Game Concept:

Idle games have been performing [extremely well recently.](https://howtomarketagame.com/2025/08/04/2025-q2-games-that-are-selling/)

## Simple idea; Idle game:
- Gain money + harvest resources
- Buy items + upgrades (skill tree)
- Unlock new stuff
- Play minigames and discover new things

The overall goal is to increase your resource production.
(Similar to cookie-clicker, Banana, Keep-on-Mining, IdleOn MMO, etc)



## MAIN CONCEPT: Upgrades + Tokens

Harvesting area is where you harvest tokens in.  
Harvesting earns money.

Upgrade-tree is where you purchase more upgrades (eg improved harvesting-speed) or, purchase 

You can upgrade harvesting-speed,
AND you can upgrade what tokens spawn. 
E.g. you can buy more exotic tokens that give good rewards,
Or you can buy extra basic tokens so theres more of them

(See `resources.md`)

## Progressive map unlocks and upgrades:
**First upgrades should be VISUAL upgrades. We need to hook the player!!!**  

Over time, player unlocks more stuff as they progress.
We also need to make the player *excited* to unlock new stuff.


## Dangled upgrades:
We really want to "dangle" upgrades in front of players.

Players should ALWAYS have multiple things to look-forward to.   
- "Just another upgrade",  
- "Once I get THIS last upgrade, it'll be great".  
^^^ it sounds bad, but that's how we need to engineer the game.  

Likewise, upgrade pacing should be decently fast.  
We dont want it to take 30 minutes to get the next upgrade. Players should get it easily.



## Prestige:
QUESTION: "Why should the player prestige?"  
-> Because it unlocks new stuff.

When the player prestiges, there should *ideally* be a clear difference in gameplay; 
Ideally, a different upgrade-path should be available instantly.




## Pacing:
Be very very careful about pacing.    
"Given an opportunity, players will optimize the fun out of the game".  
^^^ this is especially true for idle games.

Do NOT let the players abuse the systems.
- USE ANALYTICS.

We should write down exactly WHEN we expect players to get certain upgrades.  
Then, track how fasts players progress.  
Ideally, we want players to progress *steadily*, but not toooo fast.

(We should weigh exciting progress more towards the start of the game;
ie the "hook". Every minute or so when starting, a new upgrade.)

### Pacing: Upgradable Resource-limits. 
EG: money capped at $5000 (can be upgraded)
With limits, offline/idle gameplay becomes a LOT less damaging.  
(If the player idles for many hours, they don't become giga-rich.)  

*WHATEVER THE CASE MAKE SURE PLAYERS CAN ALWAYS SPEND!*    
Nothing would feel worse than being capped out and not having anything to spend it on.  
What should be done, is the resource-limit for Money should be upgraded via Money.  
Logs resource limit should be upgraded via Logs.  
Rocks resource limit should be upgraded via Rocks.  
etc. This *guarantees* that players always have stuff to spend.
